# Product

## Register

product

## Users

Developers and designers using a desktop AI app generator. They work at a workstation, often for extended sessions, iterating quickly on prototype apps. They are technical, keyboard-driven, and value speed over chrome.

## Product Purpose

Prototyper generates full application prototypes from AI prompts. The assets section generates images locally via Bonsai Image AI for use in those prototypes. Success means the user goes from idea to running prototype with minimal friction, and can manage generated assets (images) as first-class project resources alongside their code.

## Brand Personality

precise, utilitarian, software code

## Anti-references

Figma's decorative onboarding. Notion's playful illustration style. Canva's consumer-friendly gradient surfaces. Anything that looks like a marketing landing page inside a tool.

## Design Principles

1. **Tool, not gallery.** Every pixel serves a workflow. Assets are displayed for identification and action, not admiration.
2. **Dense and scannable.** Show as much useful metadata as fits without noise. Prompt, size, filename, seed: all visible at a glance.
3. **Code-native feel.** The UI should feel like a developer tool, not a consumer app. Monospace where appropriate, flat surfaces, no decorative gradients or shadows.
4. **Instrument, not canvas.** Controls are compact, labeled precisely, and grouped by function. No hero moments, no empty celebration.
5. **Keyboard-speed flow.** Every common action reachable in one click or one shortcut. Generate, navigate, delete, reveal.

## Accessibility & Inclusion

WCAG 2.1 AA minimum. High contrast in dark theme. Keyboard navigation for gallery. Focus indicators visible on all interactive elements.
